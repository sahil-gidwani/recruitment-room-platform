# Recruitment Room

Recruitment Room is a lightweight analytics workspace for exploring football player data with a focus on scouting, comparison, and decision support. This repository is intentionally minimal at the start so new contributors can understand the project direction before any feature code is added.

## Project Overview

The project will grow into a Streamlit-based platform for loading player data, visualizing performance, and supporting scouting workflows. The first milestone is documentation and structure, followed by a careful buildout of the application in small, understandable steps.

## Vision and Goals

Recruitment Room exists to make player evaluation easier, more consistent, and more accessible for recruitment teams. The long-term goal is to provide a clear workflow from raw data to scouting insight without requiring a large or complex codebase.

## Why It Exists

Scouting decisions often depend on scattered spreadsheets, manual comparisons, and inconsistent reporting. Recruitment Room is intended to bring those workflows into one place with a simple interface, transparent analysis, and room to scale over time.

## High-Level Roadmap

The project roadmap starts with foundations such as CSV upload, filtering, tables, and basic visualizations. It then expands into analytics, scouting intelligence, and automation. See the full plan in [docs/roadmap.md](docs/roadmap.md).

## Contribution Workflow

Contributions should stay small, focused, and easy to review. Start by reviewing the architecture and setup guides, claim or discuss an issue, create a short-lived branch, and open a pull request with a clear summary of what changed. See [docs/contribution_guide.md](docs/contribution_guide.md) for the full workflow.
